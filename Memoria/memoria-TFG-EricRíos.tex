\begin{abstract}
{\em
Los mundos virtuales explorables en los videojuegos constituyen el núcleo de la experiencia del jugador, interactuando e influyendo en todos los aspectos del juego. 
Tanto la mecánica de juego como los gráficos o la narrativa, cada elemento de un videojuego debe funcionar en armonía con su representación virtual del espacio. 
A medida que los juegos se vuelven más ambiciosos, surge la necesidad de crear mundos enormes y orgánicos.
Sin embargo, a la vez que aumenta el tamaño de los mundos, también lo hace el esfuerzo necesario para crearlos a mano.
Para evitar cargas de trabajo imposibles y optimizar los procesos de desarrollo, muchos estudios han adoptado la generación procedural para crear algorítmicamente los mundos de sus juegos.

El objetivo de este proyecto ha sido desarrollar un sistema de generación procedural de terrenos dentro del motor de juegos Godot, con el propósito principal de crear una herramienta de diseño útil y ampliable para el desarrollo de cualquier videojuego que requiera un mapa de gran tamaño.
El sistema se basa en un proceso de generación con varias etapas que incluye la generación de mapas de altura, la modificación del terreno basada en agentes, la colocación de elementos visuales, la gestión del nivel de detalle y la generación de colisiones, cada una de ellas diseñada para funcionar de forma modular e independiente de las demás.
Se desarrolló un marco de pruebas de rendimiento específico para evaluar el coste computacional de cada etapa del proceso.

El sistema está implementado principalmente en GDScript, con algunas partes en C++, y se emplean sombreadores (\textit{shaders}) de cómputo en GPU cuando ofrecen beneficios significativos en el rendimiento.
Toda la configuración se expone a través del sistema de recursos nativo de Godot, lo que permite a los diseñadores parametrizar y guardar perfiles de generación sin necesidad de escribir código.
La implementación completa está disponible públicamente como un repositorio de código abierto, con la intención de proporcionar una base práctica para los desarrolladores de videojuegos que necesiten generación procedural de terrenos.
}

\end{abstract}

\begin{abstract}
{\em
The virtual worlds explorable in video games are at the core of the player experience, interacting with and influencing every aspect of the game. 
From gameplay mechanics to graphics, and everything in between, each element of a video game must work in harmony with its virtual representation of space. 
As games grow more ambitious, a need for huge, organic worlds arises.
However, as the size of the worlds increases, so does the effort required to hand-craft them.
To avoid impossible workloads and streamline development processes many studios have adopted procedural generation to algorithmically generate their game worlds.

The goal of this project has been to develop a procedural terrain generation system within the Godot game engine, with the primary aim of creating a useful and extensible design tool for the development of any video game requiring a large map.
The system is built around a multi-stage generation pipeline comprising heightmap generation, agent-based terrain modification, feature placement, level of detail management, and collision generation, each designed to operate modularly and independently of the others.
A dedicated benchmarking framework was developed to evaluate the computational cost of each stage of the pipeline.

The system is implemented mainly in GDScript, with some regions in C++, and GPU compute shaders employed where they yield significant performance benefits. 
All configuration is exposed through Godot's native resource system, enabling designers to parametrize and save generation profiles without writing code. 
The complete implementation is publicly available as an open-source repository, with the intention of providing a practical foundation for game developers requiring procedural terrain generation.
}
